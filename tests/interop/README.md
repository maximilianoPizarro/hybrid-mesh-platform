# Hybrid Mesh Platform — VP Interop Tests

Tests following the [Validated Patterns](https://validatedpatterns.io) testing framework.
Executed via `make qe-tests` (VP common Makefile hook) or directly via `./run_tests.sh`.

## Test structure

| File | Scope | Description |
|------|-------|-------------|
| `test_subscription_status_hub.py` | Hub | All operator subscriptions installed and healthy |
| `test_subscription_status_edge.py` | Spoke | Spoke operator subscriptions |
| `test_validate_hub_site_components.py` | Hub | Cluster reachable, pods OK, ArgoCD apps Healthy |
| `test_validate_edge_site_components.py` | Spoke | Spoke reachable, IE + Skupper pods OK |
| `test_workshop_surfaces.py` | Hub + Spokes | HTTP 200 for all 20 console links; Kuadrant 401 |
| `test_platform_components.py` | Hub | Skupper VAN sites, GitLab Gateway, Kairos policies, Kuadrant |
| `test_modify_web_content.py` | Hub (E2E) | GitOps roundtrip: modify showroom → push → verify |

## Prerequisites

- OpenShift hub cluster with Hybrid Mesh Platform deployed
- `oc` client installed
- kubeconfig files for hub (and optionally east/west spokes)
- Python 3.9+

## Quick start

```bash
cd tests/interop

# Install dependencies
pip install -r requirements.txt

# Set required env
export KUBECONFIG=/path/to/hub-kubeconfig
export KUBECONFIG_EDGE=/path/to/east-kubeconfig   # optional
export INFRA_PROVIDER=RHDP
export HUB_APPS_DOMAIN=$(oc get ingresses.config cluster -o jsonpath='{.spec.domain}')

# Run all tests
./run_tests.sh
```

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `KUBECONFIG` | Yes | Hub cluster kubeconfig |
| `KUBECONFIG_EDGE` | No | East spoke kubeconfig |
| `KUBECONFIG_WEST` | No | West spoke kubeconfig |
| `INFRA_PROVIDER` | Yes | Cloud/infra label (RHDP, AWS, GCP, Azure) |
| `HUB_APPS_DOMAIN` | No | Auto-resolved from cluster if not set |
| `EAST_APPS_DOMAIN` | No | Auto-resolved from KUBECONFIG_EDGE if not set |
| `WORKSPACE` | No | JUnit XML output dir (default: auto-tempdir) |
| `SKUPPER_VAN_SITES` | No | Expected Skupper VAN site count (default: 3) |
| `PATTERNS_REPO_PATH` | No | Repo path for GitOps roundtrip test |
| `CONTENT_UPDATE_TIMEOUT_MINUTES` | No | GitOps roundtrip timeout (default: 10) |
| `HTTP_CHECK_TIMEOUT` | No | HTTP request timeout in seconds (default: 15) |

## Running individual tests

```bash
cd tests/interop

# Hub subscriptions only
pytest -lv test_subscription_status_hub.py --kubeconfig $KUBECONFIG

# Workshop HTTP surfaces
pytest -lv test_workshop_surfaces.py -k "developer_hub or gitlab or grafana"

# Platform components (Skupper, Kairos, Kuadrant)
pytest -lv test_platform_components.py

# All tests with JUnit output
pytest -lv --junit-xml results.xml tests/interop/
```

## Results

- JUnit XML files placed at `$WORKSPACE/*.xml`
- Test logs at `$WORKSPACE/.results/test_execution_logs/` (vp-qe-test-common)
- CI badge generated by `create_ci_badge.py`

## GitHub Actions (manual trigger)

Workflow: `.github/workflows/interop-tests.yml`

**Secrets required (in repo Settings → Secrets):**
- `HUB_KUBECONFIG_B64` — base64-encoded hub kubeconfig
- `EAST_KUBECONFIG_B64` — base64-encoded east spoke kubeconfig (optional)

**Trigger:**
```
gh workflow run interop-tests.yml \
  -f hub_domain=apps.cluster-j2j5x.dynamic2.redhatworkshops.io \
  -f infra_provider=RHDP
```
