# Support Policy

## Overview

The **Hybrid Mesh Platform** is a community-supported Validated Pattern in the **Sandbox** tier. This means it is maintained on a **best-effort basis** by the community and pattern maintainers.

## Support Tiers

| Tier | Support Level | This Pattern |
|------|---------------|--------------|
| Sandbox | Community best-effort | ✅ Current |
| Tested | Community + CI validation | Planned |
| Maintained | Red Hat supported | Future |

## What's Supported

### In Scope

- Installation and deployment guidance
- Configuration of included components
- Bug fixes for pattern-specific code
- Documentation improvements
- Community contributions via PR

### Out of Scope

- 24/7 support or SLAs
- Support for customizations outside the pattern
- Individual Red Hat product support (covered by Red Hat subscriptions)
- Production deployment consulting

## Red Hat Product Support

Individual Red Hat products included in this pattern (ACM, ACS, RHOAI, etc.) are supported through your Red Hat subscription. For product-specific issues:

1. [Red Hat Customer Portal](https://access.redhat.com)
2. [Red Hat Support Cases](https://access.redhat.com/support/cases)

## Response Times

As a community-supported project:

- **Issues**: Best-effort response, typically within 1-2 weeks
- **PRs**: Review within 1-2 weeks
- **Security issues**: Prioritized, please use private disclosure

## Security Vulnerabilities

For security vulnerabilities, please **do not** open a public issue. Instead:

1. Email the maintainers directly
2. Use GitHub's private vulnerability reporting feature
3. Allow reasonable time for a fix before public disclosure

## Disclaimer

This pattern is provided "as-is" without warranty. While we strive for quality and reliability, the community support model means:

- No guaranteed response times
- No guaranteed fixes
- No production SLAs

For production deployments requiring support guarantees, consider engaging Red Hat Professional Services or waiting for the pattern to reach "Maintained" tier.
